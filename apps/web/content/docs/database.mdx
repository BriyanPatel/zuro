<Tabs items={['npm', 'pnpm', 'bun']}>


  </Tab>
  <Tab value="pnpm">

  </Tab>
  <Tab value="bun">

```bash
bun x zuro-cli add database
```

  </Tab>
</Tabs>

You can skip the dialect prompt: `add database-pg` or `add database-mysql`.

## Usage

**1. Define a schema:**

```ts title="src/db/schema/posts.ts"
import { pgTable, text, timestamp } from "drizzle-orm/pg-core";

export const posts = pgTable("posts", {
  id: text("id").primaryKey(),
  title: text("title").notNull(),
  createdAt: timestamp("created_at").defaultNow(),
});
```

**2. Export it:**

```ts title="src/db/schema/index.ts"
export * from "./posts";
```

**3. Run migrations:**

```bash
npx drizzle-kit generate
npx drizzle-kit migrate
```

**4. Query in a route:**

```ts
import { db } from "../db";
import { posts } from "../db/schema";

router.get("/posts", async (req, res) => {
  const allPosts = await db.select().from(posts);
  res.json(allPosts);
});
```

## Configuration

Your `.env` gets a `DATABASE_URL` automatically:

```bash
# PostgreSQL
DATABASE_URL=postgresql://postgres@localhost:5432/mydb

# MySQL
DATABASE_URL=mysql://root@localhost:3306/mydb
```

| Provider    | URL Format                                                     |
| ----------- | -------------------------------------------------------------- |
| Neon        | `postgresql://user:pass@ep-xxx.neon.tech/mydb?sslmode=require` |
| PlanetScale | `mysql://user:pass@aws.connect.psdb.cloud/mydb?ssl=...`        |
| Supabase    | `postgresql://postgres:pass@db.xxx.supabase.co:5432/postgres`  |

---

## Troubleshooting

- **Invalid database URL** — must start with `postgresql://` or `mysql://`
- **Migration errors** — check that your database is running and `DATABASE_URL` is correct
- **Switching dialects** — Zuro creates a backup in `.zuro/backups/` before replacing files
