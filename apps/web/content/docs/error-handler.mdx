## Usage

**Throw typed errors in your routes:**

```ts
import { NotFoundError, ForbiddenError } from "../lib/errors";
import { asyncHandler } from "../middleware/error-handler";

router.get(
  "/users/:id",
  asyncHandler(async (req, res) => {
    const user = await db.findUser(req.params.id);

    if (!user) throw new NotFoundError("User not found");
    if (!user.isActive) throw new ForbiddenError("Account is deactivated");

    res.json(user);
  }),
);
```

The response is always consistent JSON:

```json
{
  "status": "error",
  "code": "NOT_FOUND",
  "message": "User not found"
}
```

**Available error classes:**

| Class                 | Status | Code               |
| --------------------- | ------ | ------------------ |
| `BadRequestError`     | 400    | `BAD_REQUEST`      |
| `UnauthorizedError`   | 401    | `UNAUTHORIZED`     |
| `ForbiddenError`      | 403    | `FORBIDDEN`        |
| `NotFoundError`       | 404    | `NOT_FOUND`        |
| `ConflictError`       | 409    | `CONFLICT`         |
| `ValidationError`     | 422    | `VALIDATION_ERROR` |
| `InternalServerError` | 500    | `INTERNAL_ERROR`   |

**Create your own error class:**

```ts
import { AppError } from "../lib/errors";

export class PaymentError extends AppError {
  constructor(message = "Payment failed") {
    super(message, 402, "PAYMENT_FAILED");
  }
}
```

## Configuration

No environment variables needed. Behavior adapts to `NODE_ENV` automatically:

- **development** â€” stack traces included in responses
- **production** â€” stack traces hidden, generic error messages for unexpected errors
- **Sensitive fields** (`password`, `token`, `secret`) are redacted from error logs
- **Request IDs** from `x-request-id` headers are included in responses

---

## Troubleshooting

- **Errors not being caught** â€” make sure `app.use(errorHandler)` is the last middleware in `app.ts`
- **Stack trace in production** â€” verify `NODE_ENV=production` in `.env`
- **Zod errors not formatted** â€” `ZodError` is automatically formatted as `VALIDATION_ERROR` with `422`
