# Init Command

Create a new Express.js + TypeScript project with security, logging, and environment validation pre-configured.

`init` scaffolds the **core foundation only**. Add features like auth and database later with `zuro-cli add`.
It can also set up Prettier for your generated project during the interactive flow.

---

## Quick Start

<Tabs items={['npm', 'pnpm', 'bun']}>
  <Tab value="npm">
  ```bash
  npx zuro-cli init
  ```
  </Tab>
  <Tab value="pnpm">
  ```bash
  pnpm dlx zuro-cli init
  ```
  </Tab>
  <Tab value="bun">
  ```bash
  bun x zuro-cli init
  ```
  </Tab>
</Tabs>

---

## Interactive Setup

<Steps>
<Step>
### Project Name

```
  Tip: Leave blank to use current folder (hackathon-project)

? Project name (blank for current folder) ›
```

- **Leave blank** -> Uses current folder
- **Enter name** -> Creates/uses that folder
</Step>

<Step>
### Package Manager

```
? Package Manager? ›
❯ npm
  pnpm
  bun
```

Zuro checks that your selected package manager is installed before continuing.
</Step>

<Step>
### Safety Confirmation

If target folder already exists and is not empty, Zuro asks before continuing.
</Step>

<Step>
### Optional Prettier Setup

```
? Setup Prettier? (recommended) ›
❯ Yes
  No
```

If you choose **Yes**, Zuro adds:
- `prettier` as a dev dependency
- `.prettierrc` and `.prettierignore`
- `format` and `format:check` scripts in `package.json`
</Step>

<Step>
### Done

```bash
cd my-project
npm run dev
```

Server runs at `http://localhost:3000`.
</Step>
</Steps>

## What Init Includes

| Feature | Library | Description |
|---------|---------|-------------|
| **Framework** | Express.js | Fast, minimal web framework |
| **Language** | TypeScript | Type safety out of the box |
| **Security** | Helmet | Secure HTTP headers |
| **CORS** | cors | Cross-origin requests |
| **Logging** | Pino | Fast JSON logging |
| **Env Validation** | Zod | Type-safe environment variables |
| **Routing** | Express Router | Centralized `/api` route composition |
| **Formatting (Optional)** | Prettier | Consistent formatting via init prompt |

## Safety Check

If this directory already contains a non-Zuro project, `init` stops before writing files.

Use `zuro-cli init` in:
- a fresh/empty directory, or
- an existing project already managed by Zuro CLI.

---

## Troubleshooting

- **Registry unreachable**: Verify network and registry status, then run `npx zuro-cli init` again.
- **Package manager not found**: Install the selected package manager, or re-run init and choose another one.
- **Target folder not empty**: Confirm overwrite prompt, or use a new empty folder.
