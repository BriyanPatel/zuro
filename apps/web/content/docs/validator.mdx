## Usage

Add `validate(schema)` as middleware to any route. Your schema describes what the request should look like.

**Validate a POST body:**

```ts
import { z } from "zod";
import { validate } from "../middleware/validate";

const schema = z.object({
  body: z.object({
    name: z.string(),
    email: z.string().email(),
  }),
});

router.post("/users", validate(schema), (req, res) => {
  // req.body is validated — safe to use
  res.json({ name: req.body.name });
});
```

**Validate URL params:**

```ts
const schema = z.object({
  params: z.object({
    id: z.string().uuid(),
  }),
});

router.get("/users/:id", validate(schema), (req, res) => {
  res.json({ userId: req.params.id });
});
```

**Validate query strings:**

```ts
const schema = z.object({
  query: z.object({
    page: z.coerce.number().default(1),
    limit: z.coerce.number().default(20),
  }),
});

router.get("/posts", validate(schema), (req, res) => {
  // page and limit are numbers, not strings
  res.json({ page: req.query.page, limit: req.query.limit });
});
```

**What happens on invalid input:**

```json
{
  "status": "error",
  "code": "VALIDATION_ERROR",
  "message": "Validation failed",
  "errors": [{ "path": "body.email", "message": "Invalid email" }]
}
```

## Configuration

No environment variables needed. Zod defaults and coercion work automatically — for example, `z.coerce.number()` converts the string `"5"` from a query param to the number `5`.

---

## Troubleshooting

- **422 Validation failed** — check that your request matches the schema fields and types
- **Query params are strings** — use `z.coerce.number()` or `z.coerce.boolean()` to convert them
- **Non-Zod errors** — they pass through to your error handler middleware
