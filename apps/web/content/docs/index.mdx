# Welcome to Zuro

**Stop wasting time on Express boilerplate.** `init` gives you the production-ready core (Express + TypeScript + security + logging + env validation). Then use `add` to install modules like auth, database, and validator when you need them.

```bash
npx zuro-cli init
```

Then add modules for your use case:

```bash
npx zuro-cli add auth
```

---

## How It Works

<Steps>
<Step>
### Init

```bash
npx zuro-cli init
```

Creates Express + TypeScript project with security, logging, and env validation.
</Step>

<Step>
### Add Modules

```bash
npx zuro-cli add auth
```

Add features when you need them. Dependencies are auto-installed—running `add auth` also sets up database and error-handler.
</Step>

<Step>
### Ship

```bash
npm run dev
```

Build your actual features. We handle the infrastructure.
</Step>
</Steps>

## Quick Start

<Tabs items={['npm', 'pnpm', 'bun']}>
  <Tab value="npm">
    ```bash
    # Create project
    npx zuro-cli init my-app
    cd my-app

    # Add auth (auto-installs database, error-handler)
    npx zuro-cli add auth

    # Run
    npm run dev
    ```
  </Tab>
  <Tab value="pnpm">
    ```bash
    # Create project
    pnpm dlx zuro-cli init my-app
    cd my-app

    # Add auth (auto-installs database, error-handler)
    pnpm dlx zuro-cli add auth

    # Run
    pnpm dev
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    # Create project
    bun x zuro-cli init my-app
    cd my-app

    # Add auth (auto-installs database, error-handler)
    bun x zuro-cli add auth

    # Run
    bun run dev
    ```
  </Tab>
</Tabs>

<Callout type="info">
**One command, full stack:** Running `zuro-cli add auth` automatically installs database and error-handler if they're missing. No need to remember the order!
</Callout>

## Available Modules

| Module | Command | What You Get |
|--------|---------|--------------|
| **Core** | `zuro-cli init` | Express, TypeScript, Helmet, CORS, Pino logger |
| **Database** | `zuro-cli add database` | Drizzle ORM, PostgreSQL or MySQL |
| **Auth** | `zuro-cli add auth` | Better-Auth, signup/login, sessions |
| **Validator** | `zuro-cli add validator` | Zod middleware for request validation |
| **Error Handler** | `zuro-cli add error-handler` | Custom error classes, consistent responses |

## What Makes Zuro Different?

<Accordions type="multiple">
  <Accordion title="Not a Framework">
    We don't add runtime dependencies. Zuro generates plain Express.js + TypeScript files that you own 100%. No magic, no decorators, no lock-in.

    **You can read and modify every line of code.**
  </Accordion>

  <Accordion title="Built for Speed">
    Every feature saves you time:
    - **Smart dependencies** — modules auto-install their requirements
    - Database prompts with localhost defaults
    - Auth with auto-generated secrets
    - Environment files update automatically
    - Routes inject into app.ts automatically
  </Accordion>

  <Accordion title="Learn Real Patterns">
    The generated code follows production best practices. Great for students learning backend development—you can understand every line.
  </Accordion>
</Accordions>

## Tech Stack

Everything uses battle-tested libraries:

| Category | Library | Why |
|----------|---------|-----|
| Framework | Express.js | Most popular Node.js framework |
| Language | TypeScript | Type safety out of the box |
| Database | Drizzle ORM | Type-safe queries, great DX |
| Auth | Better-Auth | Modern, flexible auth |
| Validation | Zod | Runtime + TypeScript validation |
| Security | Helmet | Secure HTTP headers |
| Logging | Pino | Fastest JSON logger |
