# auth

## Overview

Add production-ready email/password authentication and session-based user access to your backend.

---

## Install

```bash
npx zuro add auth
```

Installs Better Auth, scaffolds auth files/routes, and auto-installs required module dependencies (`database`, `error-handler`) if missing.

---

## What This Generates

```text
src/
├ lib/
│ └ auth.ts
├ controllers/
│ └ user.controller.ts
├ routes/
│ └ user.routes.ts
└ db/schema/
  └ auth.ts       # mysql projects receive mysql-compatible auth schema

src/app.ts         # updated: mounts /api/auth handler
src/routes/index.ts # updated: mounts /users routes
```

`lib/auth.ts`: Better Auth setup (`basePath: /api/auth`) with Drizzle adapter.
`controllers/user.controller.ts`: authenticated profile handler (`/api/users/me`).
`routes/user.routes.ts`: user routes wired with async error handling.
`db/schema/auth.ts`: auth tables (`user`, `session`, `account`, `verification`).
`app.ts` and `routes/index.ts`: route mounts injected automatically.

## How It Works

Client -> `/api/auth/*` (Better Auth handler) -> session/cookie issued -> `/api/users/me` -> `auth.api.getSession(...)` -> database-backed session lookup -> response

---

## Configuration

```bash
BETTER_AUTH_SECRET=your-secret-key-at-least-32-characters-long
BETTER_AUTH_URL=http://localhost:3000
DATABASE_URL=postgresql://postgres@localhost:5432/mydb
```

`BETTER_AUTH_SECRET`: signing/encryption secret for auth sessions and tokens.
`BETTER_AUTH_URL`: backend base URL used by Better Auth.
`DATABASE_URL`: Drizzle database connection for auth tables and sessions.

---

## API Reference

`POST /api/auth/sign-up/email`: register user with email/password.
`POST /api/auth/sign-in/email`: authenticate user and create session.
`POST /api/auth/sign-out`: invalidate current session.
`GET /api/auth/get-session`: read current session from cookie.
`GET /api/users/me`: return authenticated user profile from active session.

## Advanced Usage

Protect custom routes with Better Auth session checks:

```ts
import { Router } from "express";
import { fromNodeHeaders } from "better-auth/node";
import { auth } from "../lib/auth";
import { UnauthorizedError } from "../lib/errors";
import { asyncHandler } from "../middleware/error-handler";

const router = Router();

router.get("/dashboard", asyncHandler(async (req, res) => {
  const session = await auth.api.getSession({
    headers: fromNodeHeaders(req.headers),
  });

  if (!session) {
    throw new UnauthorizedError("Not authenticated");
  }

  res.json({ message: `Welcome, ${session.user.name}` });
}));
```

Run migrations after install:

```bash
npx drizzle-kit generate
npx drizzle-kit migrate
```
