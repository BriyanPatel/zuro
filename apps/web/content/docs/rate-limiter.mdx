# Rate Limiter

Protect your API from abuse with automatic request rate limiting.

Adds `express-rate-limit` middleware with sensible defaults and env-based configuration.

---

## Quick Start

<Tabs items={["npm", "pnpm", "bun"]}>
  <Tab value="npm">```bash npx zuro-cli add rate-limiter ```</Tab>
  <Tab value="pnpm">```bash pnpm dlx zuro-cli add rate-limiter ```</Tab>
  <Tab value="bun">```bash bun x zuro-cli add rate-limiter ```</Tab>
</Tabs>

That's it — no prompts, no configuration needed. Works out of the box.

- Injects `app.use(rateLimiter)` into `app.ts` (after `express.json()`)

---

## Default Limits

| Setting          | Default       | Description                        |
| ---------------- | ------------- | ---------------------------------- |
| **Window**       | 15 minutes    | Time window for counting requests  |
| **Max Requests** | 100           | Maximum requests per IP per window |
| **Headers**      | `RateLimit-*` | IETF draft-7 standard headers      |

When a client exceeds the limit, they receive:

```json
{
  "status": "error",
  "code": "RATE_LIMIT_EXCEEDED",
  "message": "Too many requests, please try again later."
}
```

---

## Configuration

Customize limits via environment variables — no code changes needed:

```env
# .env
RATE_LIMIT_WINDOW_MS=900000   # 15 minutes (in milliseconds)
RATE_LIMIT_MAX=100             # requests per window per IP
```

**Common presets:**

| Use Case     | Window | Max | Env Values      |
| ------------ | ------ | --- | --------------- |
| Public API   | 1 min  | 30  | `60000`, `30`   |
| Standard App | 15 min | 100 | `900000`, `100` |
| Internal API | 1 min  | 200 | `60000`, `200`  |

## Per-Route Rate Limiting

The default rate limiter applies globally. For stricter limits on specific routes (e.g., login), create a second limiter:

```typescript
import rateLimit from "express-rate-limit";

const authLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 10, // much stricter
  message: {
    status: "error",
    code: "RATE_LIMIT_EXCEEDED",
    message: "Too many login attempts, please try again later.",
  },
});

// Apply only to login route
router.post("/login", authLimiter, loginController);
```

---

## How It Works With Auth

If you have the **auth** module installed, both rate limiters work together:

1. **Global rate limiter** (this module) — 100 req/15 min across all routes
2. **Better Auth's built-in limiter** — additional protection on auth endpoints

This provides defense in depth — the global limiter catches broad abuse while Better Auth handles auth-specific attacks.

## Response Headers

Clients receive standard `RateLimit-*` headers on every response:

```
RateLimit-Limit: 100
RateLimit-Remaining: 97
RateLimit-Reset: 1709420400
```

These headers let frontends show "slow down" warnings before hitting the limit.

## Troubleshooting

- **Getting 429 during development?** Increase `RATE_LIMIT_MAX` in `.env` or set it to `0` to disable (not recommended for production).
- **Rate limit not working behind a proxy?** Set `app.set("trust proxy", 1)` in `app.ts` so Express reads `X-Forwarded-For` headers.
- **Need different limits per route?** See [Per-Route Rate Limiting](#per-route-rate-limiting) above.
