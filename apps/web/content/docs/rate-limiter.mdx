## Usage

Rate limiting is applied globally after install — all routes are protected automatically.

**Default behavior:** 100 requests per 15 minutes per IP. When exceeded:

```json
{
  "status": "error",
  "code": "RATE_LIMIT_EXCEEDED",
  "message": "Too many requests, please try again later."
}
```

**Add stricter limits on specific routes:**

```ts
import rateLimit from "express-rate-limit";

const loginLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 10,
  message: {
    status: "error",
    code: "RATE_LIMIT_EXCEEDED",
    message: "Too many login attempts, please try again later.",
  },
});

router.post("/login", loginLimiter, loginController);
```

---

## Configuration

Customize limits via environment variables — no code changes needed:

```bash
RATE_LIMIT_WINDOW_MS=900000   # 15 minutes in ms
RATE_LIMIT_MAX=100             # requests per window per IP
```

| Use Case     | Window | Max | Values          |
| ------------ | ------ | --- | --------------- |
| Public API   | 1 min  | 30  | `60000`, `30`   |
| Standard App | 15 min | 100 | `900000`, `100` |
| Internal API | 1 min  | 200 | `60000`, `200`  |

---

## Troubleshooting

- **Getting 429 during development** — increase `RATE_LIMIT_MAX` in `.env`
- **Not working behind a proxy** — add `app.set("trust proxy", 1)` in `app.ts`
